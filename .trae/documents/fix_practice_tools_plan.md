# 练习工具和选集问题修复计划

## 一、问题分析

### 问题1：练习工具按钮无法使用
- **现象**：练习工具面板中的播放/暂停、重置、节拍器按钮点击无反应
- **根因**：需要检查按钮是否被正确渲染，以及是否有事件绑定问题

### 问题2：选集集数错误
- **现象**：当前视频只有6集，但显示了10集（第7集及以后不存在）
- **根因**：我们之前在 defaultData 中错误地给视频添加了10集，而实际视频只有6集

### 问题3：选集标题显示为通用名称
- **现象**：选集只显示"第1集"、"第2集"等，没有显示实际分集标题
- **根因**：虽然我们添加了分集数据，但当前显示的视频可能没有对应的 episodes 数据，回退到了 fallback

## 二、修复方案

### 2.1 修复练习工具按钮

**修改文件**：
- `src/pages/VideoStudyPage.tsx` - 检查练习工具按钮的事件绑定

**具体修改**：
1. 确保所有按钮都有正确的 onClick 事件
2. 确保按钮没有被其他元素遮挡
3. 确保状态变量正确初始化

### 2.2 修复选集集数和标题

**修改文件**：
- `src/data/defaultData.ts` - 修正各视频的分集数量和标题
- `src/utils/bilibili.ts` - 修复 getBiliFallbackEpisodes 函数，避免显示不存在的集数

**具体修改**：
1. 为每个视频资源添加正确的分集数量和标题（基于实际视频）
2. 修复 fallback 逻辑，使用视频资源中定义的集数，而不是固定的10集

### 2.3 修复 VideoEpisodePicker

**修改文件**：
- `src/components/VideoPlayerCard.tsx` - 使用视频资源中的集数信息

**具体修改**：
1. 当 customEpisodes 存在时，优先使用 customEpisodes
2. 当 API 请求失败时，使用视频资源中定义的集数（如果有），而不是固定的10集

## 三、实施步骤

### 步骤1：修复练习工具按钮
1. 检查 VideoStudyPage 中练习工具按钮的事件绑定
2. 确保所有按钮都能正常工作

### 步骤2：修复选集数据
1. 更新 defaultData.ts，为每个视频添加正确的分集数据
2. 修复 bilibili.ts 中的 fallback 逻辑

### 步骤3：修复 VideoEpisodePicker
1. 更新 VideoEpisodePicker，优先使用视频资源中的集数

### 步骤4：构建验证和测试
1. 运行 npm run build
2. 测试练习工具按钮
3. 测试选集功能

## 四、风险处理

1. **API请求失败**：确保 fallback 数据也有合理的分集数量
2. **按钮功能回归**：修复后测试所有按钮功能
3. **数据不一致**：确保 defaultData 中的分集数量与实际视频一致

2. ✅ 选集显示正确的集数（不超过实际集数）
