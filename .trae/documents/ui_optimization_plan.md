# UI 分层优化计划（按优先级）

## 一、优先级说明

基于影响力、实现难度和用户价值，将 UI 优化分为三个优先级：

- **P0（高优先级）**：视觉层次优化 + 交互体验提升 — 影响整体观感和操作反馈，立竿见影
- **P1（中优先级）**：移动端适配优化 + 数据可视化增强 — 提升特定场景体验
- **P2（低优先级）**：练习沉浸模式 — 较大的功能新增，需要更多设计和开发

---

## 二、P0 高优先级：视觉层次 + 交互体验

### 目标
增强信息层级清晰度，提升操作反馈感，让界面更精致、更有生命力。

### 2.1 视觉层次优化

#### 2.1.1 卡片阴影层次系统
**当前问题**：卡片阴影只有两种（static/elevated），层次不够丰富

**优化方案**：
- 新增 4 级阴影系统：
  - `shadow-xs`：极淡阴影（用于输入框、chip等小组件）
  - `shadow-sm` / `shadow-static`：普通卡片
  - `shadow-md` / `shadow-elevated`：悬浮卡片
  - `shadow-lg` / `shadow-float`：弹窗、底部面板
- 为不同层级的内容使用对应阴影，建立清晰的 z 轴层次

**修改文件**：
- `tailwind.config.js` - 扩展 boxShadow 配置
- `src/index.css` - 更新 .glass-card 系列类名

#### 2.1.2 字体大小梯度优化
**当前问题**：字体大小跨度不够，信息层级不明显

**优化方案**：
- 建立清晰的字体层级：
  - 页面大标题：24px / 28px (font-bold)
  - 卡片标题：18px / 24px (font-bold)
  - 正文内容：14px / 22px (font-medium)
  - 辅助文字：12px / 18px (font-normal)
  - 标签/chip：11px / 16px (font-medium)
- 标题和正文的大小差从 2px 增加到 4-6px

**修改文件**：
- 各页面组件 - 统一调整字体大小
- `tailwind.config.js` - 扩展 fontSize 配置

#### 2.1.3 间距系统规范化
**当前问题**：间距使用比较随意，缺乏统一的节奏

**优化方案**：
- 建立 4px 基准的间距系统（4, 8, 12, 16, 20, 24, 32, 40, 48）
- 卡片内边距：20px（移动端）/ 24px（桌面端）
- 卡片间距：16px
- 元素间距：8px（小组件）/ 12px（中等元素）/ 16px（大模块）

**修改文件**：
- 各页面组件 - 统一调整间距

#### 2.1.4 圆角系统统一
**当前问题**：圆角大小不一致

**优化方案**：
- 卡片圆角：16px（lg）
- 按钮圆角：12px（md）
- 输入框圆角：10px
- Chip/标签圆角：999px（全圆角）
- 小图标按钮：8px

**修改文件**：
- `tailwind.config.js` - 完善 borderRadius 配置
- 各组件 - 统一圆角使用

### 2.2 交互体验提升

#### 2.2.1 按钮微动画增强
**当前问题**：按钮已有 hover 效果，但可以更精致

**优化方案**：
- 增加按钮点击时的缩放效果（scale(0.97)）
- 增加按钮的激活态背景色变化
- 为主要操作按钮添加微妙的呼吸光效（可选）
- 增加 ripple 点击波纹效果（可选，轻量级）

**修改文件**：
- `src/index.css` - 增强 .btn-primary / .btn-secondary 动画
- 或创建 `src/components/Button.tsx` 统一按钮组件

#### 2.2.2 页面切换过渡动画
**当前问题**：页面切换是硬切，没有过渡效果

**优化方案**：
- 页面切换时添加淡入 + 轻微上移效果（fade + slide-up）
- 使用 React 的 key 触发动画重渲染
- 动画时长：200-300ms，ease-out 缓动

**修改文件**：
- `src/App.tsx` - 添加页面过渡动画
- `src/index.css` - 添加过渡动画类

#### 2.2.3 卡片悬停效果
**当前问题**：可点击的卡片没有 hover 反馈

**优化方案**：
- 可交互卡片（视频卡片、知识条目等）添加 hover 效果：
  - 轻微上移（translateY(-2px)）
  - 阴影增强
  - 边框颜色轻微加深
- 动画时长：200ms

**修改文件**：
- `src/components/GlassCard.tsx` - 添加 hover 可选属性
- 各卡片组件 - 应用 hover 效果

#### 2.2.4 底部导航激活动画
**当前问题**：底部导航切换只有颜色变化

**优化方案**：
- 激活状态添加轻微放大效果（scale(1.1)）
- 图标 strokeWidth 变化（2 → 2.5，已有）
- 添加底部指示器的平滑过渡（可选）

**修改文件**：
- `src/components/BottomNav.tsx` - 增强激活态动画

## 三、P1 中优先级：移动端适配 + 数据可视化

### 3.1 移动端适配优化

#### 3.1.1 触摸目标区域优化
**当前问题**：部分按钮/可点击区域可能小于 44x44px

**优化方案**：
- 所有可交互元素最小触摸区域：44x44px
- 底部导航按钮高度：48px+
- 小图标按钮增加 padding 扩大点击区域
- 检查所有 chip、标签的点击可用性

**修改文件**：
- `src/components/BottomNav.tsx`
- 各页面中的小按钮、图标按钮

#### 3.1.2 刘海屏 / 安全区域适配
**当前问题**：底部导航可能被刘海屏遮挡

**优化方案**：
- 使用 `env(safe-area-inset-bottom)` 适配底部安全区域
- 顶部页面标题也考虑安全区域
- 底部弹窗（BottomSheet）适配安全区域

**修改文件**：
- `src/index.css` - 添加安全区域工具类
- `src/components/BottomNav.tsx`
- `src/components/PageShell.tsx`
- `src/components/QuickAddMaterialSheet.tsx`

#### 3.1.3 横屏模式优化
**当前问题**：横屏时底部导航占空间，内容区域高度不足

**优化方案**：
- 横屏时底部导航改为侧边导航或隐藏
- 练习页横屏时优先显示 BPM 旋钮和计时器
- 减少垂直方向的 padding

**修改文件**：
- `src/App.tsx` - 检测横竖屏
- `src/components/BottomNav.tsx` - 横屏适配
- 各页面组件 - 横屏样式调整

### 3.2 数据可视化增强

#### 3.2.1 练习时长趋势图（迷你图）
**当前问题**：只有数字统计，没有趋势可视化

**优化方案**：
- 在今日页 / 我的页添加 7 天练习时长迷你柱状图
- 使用纯 CSS 或 SVG 实现，不需要图表库
- 柱子高度对应练习时长，hover 显示具体数值

**修改文件**：
- 新增 `src/components/MiniBarChart.tsx`
- `src/pages/TodayPage.tsx` - 添加趋势图
- `src/pages/MePage.tsx` - 添加趋势图

#### 3.2.2 Cover 进度环形图
**当前问题**：Cover 进度只有进度条，不够直观

**优化方案**：
- 在 Cover 详情页添加环形进度图
- 使用 SVG 实现，带渐变填充
- 中心显示进度百分比

**修改文件**：
- 新增 `src/components/CircleProgress.tsx`
- `src/pages/CoverPage.tsx` - 集成环形进度

#### 3.2.3 BPM 提升曲线
**当前问题**：BPM 进步只有数字，没有趋势感

**优化方案**：
- 在 AI 反馈页 / 复盘页添加 BPM 趋势折线图（迷你）
- 展示最近 5-10 次练习的最高干净 BPM
- 用不同颜色区分尝试 BPM 和干净 BPM

**修改文件**：
- 新增 `src/components/MiniLineChart.tsx`
- `src/pages/AIFeedbackPage.tsx` - 集成趋势图
- `src/pages/ReviewPage.tsx` - 集成趋势图

## 五、实施策略

### 分阶段实施
1. **第一阶段（P0）**：视觉层次 + 交互体验 — 预计改动量中等，效果最明显
2. **第二阶段（P1）**：移动端适配 + 数据可视化 — 改动量较大，需要新增组件
3. **第三阶段（P2）**：练习沉浸模式 — 功能新增，需要较多设计和开发

### 风险处理
1. **样式回归**：每完成一个模块，在多个页面验证效果
2. **性能影响**：动画使用 CSS transform/opacity，避免重排重绘
3. **兼容性**：优先使用广泛支持的 CSS 属性，安全区域适配做降级处理
