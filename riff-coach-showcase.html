为什么做这个？

作为一个自学吉他和 ukulele 的人，我发现最大的问题不是没有教程——B站上有无数好视频—— 而是没有结构化的练习追踪和智能计划。

看了很多教程，但不知道自己练了什么、练了多久、哪里卡住了。 RiffCoach 就是为了填补这个空白：AI 帮你把视频教程和 cover 目标拆成每天能练的任务， 根据你的可用时间、练习历史、BPM 和卡点，生成最小有效练习计划。

AI 高效练习计划

根据今日可用时间、精力状态、Cover 目标和最近卡点，生成最小有效练习计划

智能规则

  • 少于 15 分钟：只安排一个最重要卡点
  • 20-30 分钟：热身 + 卡点专项 + 串联 + 复盘
  • 超过 40 分钟：完整训练 + 录音 + 复盘
  • 状态很累/手指酸：不建议冲 BPM
  • 有 Cover 项目：优先选择最高进度未完成段落

输出内容

  • 今日最小有效目标
  • 为什么今天练这个
  • 分步骤计划（时长 + BPM + 说明）
  • 今天不建议做什么
  • 完成标准

完整使用流程

1
添加资料

粘贴 B站链接或标题，选择用途（关联 Cover / 收藏 / 基础练习），快速加入素材篮。

2
AI 整理

AI 根据标题自动分析，生成 2-4 个建议练习任务，确认后进入已整理练习。

3
生成计划

今日页选择可用时间（10/20/30/45分钟）和精力状态，AI 生成最小有效练习计划。

4
开始练习

进入练习页，计时器 + 节拍器 + BPM + 卡点记录 + 备注，沉浸式练习。

5
AI 反馈

完成练习后，AI 给出今日总结、尝试 BPM、最高干净 BPM、下次计划和 Cover 进度更新。

6
周复盘

查看本周练习数据，追踪 Cover 进度，分析反复卡点，调整下周计划。

技术架构

Vite 5
Vite + React 18 + TypeScript

现代化前端工程化，极速 HMR，类型安全，ES Modules。

Zustand
Zustand 状态管理

轻量级状态管理 + LocalStorage 持久化，支持旧数据迁移。

Tailwind
Tailwind CSS 3

原子化 CSS，自定义主题变量（浅紫/奶油/薄荷绿），玻璃拟态设计系统。

Audio
Web Audio API

OscillatorNode 生成节拍器声音，零音频文件依赖。

API
B站 API 集成

BVID 解析、播放器 URL 构建、选集获取（多代理 CORS 兜底）。

AI Mock 引擎

所有 AI 功能使用本地规则 Mock，代码结构支持后续替换为真实模型 API。

// 核心类型 Lesson { id, title, targetBPM, instrument, bvid, projectId, sectionId } Session { id, lessonId, bpm, cleanBPM, painPoints, painPointDetails, aiFeedback } CoverProject { id, title, sections: [{ targetBPM, currentCleanBPM, progress }] } MaterialInboxItem { id, title, type, status, extracted: { suggestedTasks } } VideoResource { id, title, bvid, skills, summary, suggestedPractice, episodes } EfficientPracticePlan { target, reason, steps, avoid, completion }

所有数据存储在 localStorage，支持旧版本数据自动迁移。

快速开始

# 克隆仓库
git clone https://github.com/asandstar/RiffCoach.git
cd RiffCoach
# 安装依赖
npm install
# 启动开发服务器
npm run dev
# 构建生产版本
npm run build

数据存储在浏览器 LocalStorage，无需后端服务，隐私安全。
